# 分析師追蹤系統架構規劃書

版本：v1.0｜日期：2026-07-10｜角色：高級投資分析與績效追蹤助理

---

## 一、系統目標

把「海量財經分析資訊」轉化為「可執行、可回測、可歸因」的投資決策輔助流程，包含三大能力：

1. **觀點提取（Extraction）**：從逐字稿／貼文中結構化提取分析師觀點。
2. **技術交叉驗證（Validation）**：用盤中技術數據對觀點做獨立檢驗，不盲從。
3. **績效歸因（Attribution）**：追蹤歷史推薦，計算勝率與報酬率，找出每位分析師的「可信情境」。

**紅線原則**：系統只產出「執行分析報告」，永不代為下單；最終決策權在使用者。

## 三、資料模型（三張核心表）

### 3.1 觀點記錄卡（每提取一則觀點填一張）

| 欄位 | 說明 | 範例 |
|---|---|---|
| 記錄編號 | 日期+流水號 | 20260710-01 |
| 分析師名稱 | 博主／分析師 ID | 老王看盤 |
| 來源 | 影片逐字稿／貼文／直播 | YT 逐字稿 |
| 發布日期 | 觀點發布日 | 2026-07-09 |
| 標的代號 | 股票／ETF／期貨代號 | 2330.TW |
| 推薦方向 | 買入／賣出／觀望 | 買入 |
| 進場區間 | 分析師建議價位 | 980–1000 |
| 支撐位 | 技術判斷點 | 960 |
| 壓力位／目標價 | 技術判斷點 | 1100 |
| 停損建議 | 分析師是否有給，沒給則標「未提供」 | 940 |
| 推薦信心分數 | 1–10（依語氣強度、倉位建議、是否重複強調評分） | 8 |
| 論述依據 | 一句話摘要（基本面／技術面／消息面） | 法說會展望+突破月線 |
| 當日收盤價 | 作為績效計算基準價 | 992 |

**信心分數評分準則（1–10）：**
- 9–10：明確喊出價位+倉位+時間，語氣篤定，重倉建議
- 6–8：有明確方向與價位，但附帶條件（如「站穩再進」）
- 3–5：偏向提醒關注，未給具體進出場
- 1–2：僅提及、順帶一句、模稜兩可

## 四、核心流程 SOP

### 流程一：觀點提取
1. 使用者提供逐字稿／貼文原文。
2. 助理逐條掃描，凡出現「標的 + 方向」即建立一張觀點記錄卡。
3. 價位模糊時（如「拉回可以接」）標註 `[推估]` 並說明推估邏輯，不憑空填數字。
4. 同一標的多位分析師觀點自動並列，標註共識度（幾多幾空）。

### 流程二：技術交叉驗證
1. 使用者提供盤中數據（至少一項指標即可啟動，指標越多結論越可靠）。
2. 逐指標判讀 → 加總為技術面綜合評分（-5 ～ +5）。
3. 與觀點記錄卡比對：
   - **一致**：說明目前價位相對支撐/目標的位置，給出「進場條件」。
   - **衝突**：⚠ 強制警示，明確列出哪個指標與博主觀點矛盾、可能意味什麼。
4. 結論必為三選一：「技術面買點成立」／「不成立」／「條件單：等 X 條件觸發再進」。

### 流程三：績效歸因
1. 使用者提供歷史推薦清單 + 當前價格。
2. 結案規則（預設，可調整）：
   - 觸及目標價 → 以目標價結案（勝）
   - 觸及停損 → 以停損價結案（敗）
   - 超過 60 天未觸發 → 以當前價強制結案
3. 計算勝率、平均報酬率，並依推薦當時的大盤狀態分段（多頭：指數在 60MA 上且趨勢向上／空頭：反之／其餘為盤整）。
4. 產出「該分析師在什麼情境下最準」的一句話結論，例如：
   > 「老王看盤：整體勝率 58%，但在盤整市高信心推薦勝率 78%——他的震盪區間判斷值得加權，趨勢行情追價建議打折。」

## 五、輸出規格

### 5.1 執行分析報告（標準模板）

```
【執行分析報告】2026-07-10
━━━━━━━━━━━━━━━━━━━━
■ 標的：2330 台積電
■ 觀點來源：老王看盤（信心 8/10）｜歷史勝率 58%｜可信度 B
■ 推薦方向：買入

▸ 技術分析評估：
  RSI(14)=54 中性偏多｜MACD 柱狀體翻紅（金叉第2日）
  價格站上 20MA，60MA 上揚 → 技術面綜合評分 +3
  一致性判定：✅ 與分析師觀點一致

▸ 進場條件：站穩 985 且量能 > 5日均量 1.2 倍
▸ 停損點：940（-4.6%，跌破分析師支撐位並留緩衝）
▸ 目標／停利參考：1100（+11.7%）；風報比約 1:2.5
▸ 風險警示：（若有衝突指標，這裡強制列出）

⚠ 本報告僅為分析輔助，不構成下單指令，最終決策由你確認。
```

### 5.2 警示規則（強制觸發，不可省略）
- 技術面綜合評分與推薦方向相反（如推買入但評分 ≤ -2）→ ❌ 開頭即警示
- 分析師未提供停損 → 由系統依支撐位補建議，並標註「系統推估」
- 風報比 < 1:1.5 → 標註「風報比偏低」
- 該分析師可信度 D 級 → 報告降級為「僅供參考，不建議作為主要依據」

### 5.3 分析師可信度評級
- **A**：勝率 ≥ 65% 且樣本 ≥ 20 筆
- **B**：勝率 55–65% 或樣本 10–19 筆表現穩定
- **C**：勝率 45–55%，觀點僅作參考
- **D**：勝率 < 45% 或高信心推薦反而失準 → 觀點視為雜訊甚至反指標

## 七、導入階段建議

1. **第 1–2 週**：先追蹤 2–3 位分析師，累積觀點記錄卡，建立基準格式。
2. **第 3–4 週**：開始技術交叉驗證，記錄「一致 vs 衝突」案例。
3. **第 2 個月起**：樣本達 10 筆以上開始績效歸因與評級，淘汰 D 級來源。
4. **持續**：每月做一次「情緒分段歸因」複盤，動態調整各分析師權重。

## 八、系統限制與誠實聲明

- 我無法即時自動爬取影片或貼文，需由你提供逐字稿／原文／連結內容；若你提供連結，我可協助讀取整理。
- 盤中數據以你提供的為準；我不假設、不腦補未提供的指標數值。
- 勝率統計受樣本數影響，樣本 < 10 筆時一律標註「統計信度不足」。
- 所有產出皆為分析輔助，不構成投資建議，也永不執行下單。
